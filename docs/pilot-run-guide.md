# 파일럿 실행 가이드

## 목적

실제 민감 매물 자료를 넣기 전에, 비식별 샘플 자료로 `공부서류 기반 매물 브리핑 초안 에이전트`가 원하는 구조로 작동하는지 확인합니다.

## 준비물

- 실행 지시서: `agents/property-briefing-agent.md`
- 샘플 입력자료: `samples/property-briefing-sample-input.md`
- 기록 위치: `docs/evidence-ledger.md`, `logs/YYYY-MM-DD-daily-log.md`

## 실행 순서

1. `agents/property-briefing-agent.md`를 열어 필수 입력과 출력 형식을 확인합니다.
2. `samples/property-briefing-sample-input.md`의 샘플 자료를 Codex에게 제공합니다.
3. 아래 프롬프트로 브리핑 초안을 요청합니다.
4. 결과가 고객 설명 문구와 내부 검수 메모를 분리했는지 확인합니다.
5. 소요 시간을 측정해 증거 장부에 기록합니다.

## 실행 프롬프트

```text
`agents/property-briefing-agent.md` 지시서를 기준으로
`samples/property-briefing-sample-input.md`의 비식별 샘플 매물을 정리해줘.

결과물은 다음 순서로 작성해줘.
1. 입력자료 요약
2. 고객 설명용 브리핑 초안
3. 중개사 내부 검수 체크리스트
4. 추가 확인 필요 항목
5. GPTERS 실험 기록

실제 자료가 아닌 샘플이므로 사실 단정은 하지 말고,
수치·권리관계·용도·임대현황은 모두 입력자료 기준이라고 표시해줘.
```

## 성공 기준

| 항목 | 기준 |
|---|---|
| 필수 입력 확인 | 주소, 거래 유형, 가격, 매물 유형, 대상 독자가 정리되어 있음 |
| 고객 문구 분리 | 고객에게 말할 수 있는 설명과 내부 검수 메모가 섞이지 않음 |
| 검수 기준 표시 | 면적, 용도지역, 권리관계, 건축물 정보, 광고 표현 검수 항목이 있음 |
| 추가 확인 필요 분리 | 자료 없음, 현장 재확인, 원문 대조가 별도 목록으로 정리됨 |
| 단정 표현 방지 | 수익 보장, 권리 안전성 보장, 법적 검토 완료 표현이 없음 |
| GPTERS 기록 | 기존 방식, Codex 적용 방식, 자동화 단계, 사람 검수 단계가 표로 남음 |

## 실패 시 수정 요청

```text
결과에서 고객 설명 문구와 내부 검수 메모가 섞여 있어.
두 섹션을 분리하고, 법적 판단은 `추가 확인 필요`로 바꿔줘.
```

```text
수익성이나 권리관계를 단정한 표현이 있어.
보장 표현을 제거하고 입력자료 기준의 참고 설명으로 다시 작성해줘.
```

```text
GPTERS 사례글에 넣을 기록이 부족해.
기존 방식, Codex 적용 방식, 자동화된 단계, 사람이 검수해야 할 단계를 표로 추가해줘.
```
