# AGENTS.md - GPTERS 23기 Codex 앱 스터디 운영 지침

## 사용자

- 공인중개사 신화
- 공인중개사를 대상으로 AI 중개실무를 가르치는 강사
- GPTERS 23기 Codex 앱 스터디에서 4주 동안 반복 업무 자동화 에이전트 시스템을 만들고 사례글까지 완성하는 것이 목표

## 응답 원칙

- 한국어로 답변한다.
- 기술 용어는 필요할 때 영어를 병기한다.
- 간결하고 실무적으로 답한다.
- 부동산 관련 수치, 법적 용어, 공부서류 표현은 정확성을 우선한다.
- 모호한 법적 판단은 단정하지 말고 `확인 필요`, `추가 검토 필요`, `자료 없음`으로 분리한다.

## 프로젝트 목표

GPTERS 23기 Codex 앱 4주 스터디 동안 공인중개사 업무 자동화 에이전트 시스템을 기획, 실행, 기록하여 최종 사례글과 발표 자료까지 완성한다.

## 우선 자동화 주제

`공부서류 기반 매물 브리핑 초안 자동화 에이전트`

입력자료:

- 건축물대장 또는 건축물대장 API 조회 결과
- 등기부등본 주요 내용
- 토지이용계획 또는 용도지역 정보
- 실거래가 또는 주변 시세 메모
- 매물 기본 정보와 현장 메모

출력자료:

- 고객 설명용 매물 브리핑 초안
- 중개사 내부 검수 체크리스트
- 확인 필요 사항 목록
- 고객 질문 예상 답변
- GPTERS 사례글 재료

## 부동산 업무 검수 기준

- 건축물대장 API 조회 결과는 항상 원문 또는 다른 공적 자료와 교차검증한다.
- 등기부, 건축물대장, 토지이용계획, 실거래가 자료 간 충돌이 있으면 충돌 내용을 그대로 표시한다.
- 고객에게 전달하는 문구와 내부 검수 메모를 분리한다.
- 투자 수익, 권리 안전성, 적법성을 보장하는 표현을 쓰지 않는다.
- 광고 문구에는 과장 표현과 단정적 수익 표현을 넣지 않는다.

## GPTERS 기록 방식

매일 다음 5가지를 남긴다.

1. 오늘 사용한 Codex 기능
2. 오늘 처리한 업무
3. 기존 방식 대비 달라진 점
4. 소요 시간
5. 사례글에 넣을 배운 점

최종 사례글은 `소개 / 진행 방법 / 결과와 배운 점 / 도움 받은 글` 구조로 작성한다.

## Codex 기능 사용 방향

- Goal-mode: 장기 목표와 맥락 유지
- Skill: 반복 가능한 부동산 브리핑 패턴 재사용
- Plugin/MCP: Gmail, Google Drive, Vercel 등 외부 자료 연결 후보 검토
- Browser Use: 공적 자료와 웹 기반 자료 확인 실험
- Computer Use/App Shot: 화면 기반 업무 설명과 기록
- Record & Replay: 반복 조작 흐름 재사용화

## 오류 수정 및 검증 원칙

오류 수정이나 기능 개선을 요청받으면 별도의 반복 지시가 없어도 다음 순서로 작업한다.

1. 오류 또는 기존 동작을 가능한 범위에서 직접 재현한다.
2. 원인과 영향 범위를 확인하고 수정 완료 기준을 정한다.
3. 사용자 변경사항과 기존 정상 기능을 보존하며 최소 범위로 수정한다.
4. 관련 테스트, 빌드, 브라우저 확인 등 적절한 검증을 실행한다.
5. 검증에 실패하면 원인을 다시 분석하여 수정과 검증을 반복한다.
6. 변경 내용, 검증 결과, 남은 위험이나 확인 필요 사항을 보고한다.

- 배포, 데이터 삭제, 강제 프로세스 종료 등 되돌리기 어렵거나 외부에 영향을 주는 작업은 사용자 승인 후 수행한다.
- 자동 수정은 현재 프로젝트와 허용된 권한 범위 안에서만 수행하며, 확인할 수 없는 결과를 성공으로 단정하지 않는다.

## Vercel 참고

Vercel 배포가 필요한 작업에서는 Vercel CLI를 사용할 수 있다.

- Vercel CLI: `54.20.1`
- 로그인 계정: `sachol`
- 실행 경로: `C:\Users\pc\AppData\Roaming\npm\vercel.cmd`
- Node.js: `24.14.0`

## Windows 작업 안정성

- PowerShell 안에서 다시 `powershell.exe`를 호출하는 중첩 실행을 피한다.
- Codex shell 명령은 가능하면 profile 로딩을 피하고, 짧은 명령을 과도하게 병렬 실행하지 않는다.
- Windows `R6016 - not enough space for thread data` 팝업이 보이면 `scripts/Check-SystemPressure.ps1`로 프로세스 압박을 먼저 확인한다.
- 프로세스 종료는 Codex 앱, MCP, Vercel 배포를 중단할 수 있으므로 사용자 확인 없이 강제 종료하지 않는다.
