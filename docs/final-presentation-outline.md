# 최종 발표 구성안

## 발표 제목 후보

공인중개사 신화의 Codex 앱 업무 자동화 실험: 공부서류 기반 매물 브리핑 초안 에이전트 만들기

## 3분 발표 구조

| 시간 | 내용 | 핵심 메시지 |
|---|---|---|
| 0:00~0:30 | 자기소개와 문제 정의 | 반복되는 매물 브리핑 준비 시간을 줄이고 싶었다 |
| 0:30~1:10 | 기존 방식 | 공부서류, 시세, 현장 메모를 따로 확인하고 직접 초안 작성 |
| 1:10~2:00 | Codex 적용 방식 | Goal-mode, Skill, Vercel, 기록 템플릿으로 업무 흐름을 구조화 |
| 2:00~2:40 | 결과 | 브리핑 초안, 검수 체크리스트, 사례글 기록 체계 확보 |
| 2:40~3:00 | 배운 점과 다음 계획 | 사람은 판단과 검수, Codex는 정리와 초안 |

## 슬라이드 6장 구성

| 슬라이드 | 제목 | 내용 |
|---|---|---|
| 1 | 왜 시작했나 | 공인중개사 업무에서 반복되는 문서 정리와 브리핑 초안 작성 |
| 2 | 기존 업무 방식 | 입력자료, 처리 단계, 소요 시간 |
| 3 | Codex로 바꾼 방식 | Goal-mode, 문서 폴더, 작업지시서, Skill/Plugin/MCP 후보 |
| 4 | 만든 시스템 | 매물 브리핑 에이전트의 입력, 처리 흐름, 출력 |
| 5 | Before/After | 시간 비교, 줄어든 단계, 남은 검수 단계 |
| 6 | 앞으로의 개선 | Browser Use, Computer Use, Record & Replay 확장 |

## 발표에서 보여줄 산출물

- `docs/agent-system-spec.md`: 에이전트 시스템 설계
- `templates/property-briefing-intake.md`: 매물 자료 입력 체크리스트
- `docs/evidence-ledger.md`: 실제 명령과 실험 기록
- `output/study-dashboard/index.html`: 4주 운영 상태 대시보드
- `output/final-presentation/index.html`: 3분 발표용 HTML 슬라이드
- `output/final-presentation/speaker-notes.md`: 3분 발표 대본 초안
- 최종 사례글 초안: `output/gpters-case-study/post.md` 예정

## 피해야 할 표현

- `Codex가 법적 검토를 완료했다`
- `수익이 보장된다`
- `권리관계 문제가 없다`
- `건축물대장 API만으로 확정했다`

## 권장 표현

- `입력자료 기준으로 확인한 내용입니다`
- `추가 확인이 필요한 항목입니다`
- `중개사가 최종 검수할 체크리스트입니다`
- `고객 설명 전 원문 공부서류와 교차검증이 필요합니다`
