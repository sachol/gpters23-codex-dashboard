# 반복 업무 5개 입력 가이드

## 목적

Week 1 과제에서 가장 중요한 실제 입력은 `최근 반복 업무 5개`와 `각 업무의 소요 시간`입니다. 이 파일을 채우면 자동화 주제 선정, Before/After 실측, GPTERS 사례글의 정량 근거가 생깁니다.

## 입력 파일

`data/repeated-work-items.csv`

## 컬럼 설명

| 컬럼 | 적을 내용 | 예시 |
|---|---|---|
| `id` | 번호 | 1 |
| `work_item` | 반복 업무명 | 공부서류 기반 매물 브리핑 초안 |
| `current_process` | 현재 처리 방식 | 건축물대장, 등기부, 시세를 각각 확인하고 직접 정리 |
| `minutes_per_run` | 회당 소요 시간, 분 단위 | 90 |
| `input_materials` | 입력자료 | 건축물대장, 등기부등본, 현장 메모 |
| `output_artifact` | 결과물 | 고객 설명용 브리핑 초안 |
| `why_automate` | 자동화하고 싶은 이유 | 반복 시간이 길고 검수 항목이 많음 |
| `public_case_ok` | GPTERS 사례 공개 가능 여부 | yes / no / de-identify |
| `notes` | 기타 메모 | 실제 주소는 비식별 처리 필요 |

## 작성 기준

- 정확한 실측 시간이 아니면 `대략 30분`, `1시간 정도`처럼 적어도 됩니다.
- 실제 매물 주소, 고객명, 연락처, 소유자명은 공개용 자료에 넣지 않습니다.
- 부동산 관련 업무는 `고객에게 전달할 문구`와 `내부 검수 메모`를 분리할 수 있는지 함께 봅니다.
- 공개 가능 여부는 `yes`, `no`, `de-identify` 중 하나로 적습니다.

## 입력 후 할 일

1. `scripts/Test-RepeatedWorkInput.ps1`로 5개 입력 여부를 확인합니다.
2. `docs/automation-priority-matrix.md`의 후보 점수표를 실제 업무 기준으로 수정합니다.
3. `output/week1-submission/week1-assignment.md`에 실제 반복 업무와 소요 시간을 반영합니다.
4. `docs/before-after-measurement-protocol.md` 기준으로 선택 업무 1건을 실측합니다.
