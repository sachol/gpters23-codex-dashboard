# 에이전트 시스템 설계 초안

## 이름

공부서류 기반 매물 브리핑 초안 자동화 에이전트

## 목표

공인중개사가 매물 상담이나 투자 검토 전에 확인하는 자료를 Codex가 먼저 읽고, 사람이 검수하기 쉬운 브리핑 초안과 체크리스트를 만듭니다.

## 입력자료

- 건축물대장 또는 건축물대장 API 조회 결과
- 등기부등본 주요 내용
- 토지이용계획 또는 용도지역 정보
- 실거래가 또는 주변 시세 메모
- 매물 기본 정보: 주소, 면적, 가격, 층수, 주차, 방향, 임대 현황
- 현장 메모: 장점, 단점, 수리 상태, 고객 질문

## 출력자료

- 고객 설명용 매물 브리핑 초안
- 중개사 내부 검수 체크리스트
- 확인 필요 사항 목록
- 고객 질문 예상 답변
- 블로그 또는 사례글로 전환 가능한 요약

## 처리 흐름

1. 입력자료 목록과 누락 자료를 확인합니다.
2. 자료별 핵심 수치를 추출합니다.
3. 서로 충돌하는 정보와 추가 확인이 필요한 항목을 분리합니다.
4. 고객에게 설명 가능한 장점, 유의점, 질문 대응 문구를 작성합니다.
5. 중개사가 최종 확인해야 할 법적·수치적 체크리스트를 만듭니다.
6. 소요 시간과 시행착오를 기록해 GPTERS 사례글 재료로 저장합니다.

## Codex 기능 조합

- Goal-mode: 4주 맥락 유지와 장기 산출물 관리
- Skill: `/부동산-매물-브리핑`으로 브리핑 문서 작성 패턴 재사용
- MCP 또는 Plugin: Google Drive, Gmail, Vercel 등 외부 자료 연동 후보 검토
- Browser Use: 공적 자료, 실거래가, 지도 정보 확인 흐름 실험
- Computer Use 또는 App Shot: 화면 기반 업무를 Codex에게 설명하고 기록
- Record & Replay: 반복 입력/확인 흐름을 재사용 가능한 절차로 변환

## 검수 기준

- 주소, 면적, 가격, 용도지역, 건축물 용도, 층수, 사용승인일 등 핵심 수치가 입력자료와 일치해야 합니다.
- 건축물대장 API 결과는 원문 또는 다른 공적 자료와 교차검증해야 합니다.
- 법적 판단은 단정하지 않고 `확인됨`, `추가 확인 필요`, `자료 없음`으로 구분합니다.
- 고객 문구는 과장 광고, 수익 보장, 단정적 위험 축소 표현을 피합니다.

## 최종 발표용 Before/After 측정

| 항목 | 기존 방식 | Codex 적용 후 |
|---|---|---|
| 자료 확인 | 수동 확인 | 자료 목록화와 누락 확인 자동화 |
| 브리핑 초안 | 직접 작성 | 초안 생성 후 검수 |
| 체크리스트 | 경험 기반 메모 | 항목별 검수표 생성 |
| 사례글 기록 | 사후 기억 의존 | 실습 당일 로그 기반 작성 |

## 반복 실행 프롬프트 초안

```text
이 매물 자료를 공인중개사 실무 기준으로 검토해줘.
목표는 고객 설명용 브리핑 초안과 중개사 내부 검수 체크리스트를 만드는 것이다.

반드시 다음 기준을 지켜줘.
1. 주소, 면적, 가격, 용도, 층수, 사용승인일 등 수치는 입력자료 기준으로만 작성한다.
2. 자료 간 충돌이나 누락은 단정하지 말고 `추가 확인 필요`로 분리한다.
3. 건축물대장 API 결과는 원문 또는 다른 공적 자료와 교차검증이 필요하다고 표시한다.
4. 고객에게 전달할 문구와 내부 검수 메모를 분리한다.
5. 마지막에 이번 작업의 소요 시간, 자동화된 단계, 사람이 검수해야 할 단계를 표로 정리한다.
```
