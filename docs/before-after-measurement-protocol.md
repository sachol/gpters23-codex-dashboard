# Before/After 실측 프로토콜

## 목적

GPTERS 사례글과 발표 자료에 사용할 수 있도록, 기존 방식과 Codex 적용 방식의 시간을 같은 기준으로 측정합니다.

## 측정 원칙

- 같은 유형의 업무를 기준으로 비교합니다.
- `대략 빨라졌다`가 아니라 단계별 시간을 기록합니다.
- 업무 난이도가 다르면 비교표에 난이도 차이를 적습니다.
- 고객에게 전달하기 전 최종 검수 시간은 별도 항목으로 분리합니다.
- 실제 매물 자료를 사용할 경우 공개 사례글에는 주소, 소유자, 연락처 등 식별정보를 제거합니다.

## 측정 대상 업무

1차 측정 대상은 `공부서류 기반 매물 브리핑 초안 작성`입니다.

측정할 결과물:

- 입력자료 요약표
- 고객 설명용 브리핑 초안
- 중개사 내부 검수 체크리스트
- 추가 확인 필요 항목
- GPTERS 실험 기록

## 품질 검수표

속도가 빨라져도 품질이 떨어지면 자동화 성공으로 보지 않습니다.

| 검수 항목 | 기존 방식 | Codex 적용 방식 | 통과 기준 |
|---|---|---|---|
| 핵심 수치 정확성 |  |  | 입력자료와 일치 |
| 고객 설명 문구 품질 |  |  | 이해하기 쉽고 과장 없음 |
| 내부 검수 항목 누락 여부 |  |  | 주소, 면적, 용도, 권리관계, 광고 표현 포함 |
| 추가 확인 필요 항목 분리 |  |  | 자료 없음과 확인 완료를 혼동하지 않음 |
| 사례글 기록 가능성 |  |  | 시간, 입력, 출력, 배운 점 기록 |

## 실측 후 사례글에 넣을 문장 틀

```text

Codex 앱을 적용한 뒤에는 입력자료 준비 [분], 초안 생성 [분], 검수 [분]으로 나뉘었고 전체 시간은 [분]이었습니다.
가장 크게 줄어든 단계는 [단계명]이었고, 여전히 사람이 직접 확인해야 하는 단계는 [검수 항목]이었습니다.
```
