# GPTERS 23기 Codex 스터디 — Week 1 Handoff

작성일: 2026-07-22  
작성자: 도시아재

## 1. 이 문서의 목적

내일 회사 PC의 Codex 프로젝트에서 오늘의 학습 맥락과 결정 사항을 빠르게 복원하고, 첫 실무 실험을 시작하기 위한 인수인계 문서다.

## 2. 학습 배경과 목표

- 도시아재는 부동산 중개·경매 업무를 하고 있으며, 학원에서 공인중개사 대상 AI 활용 특강도 진행한다.
- 이번 GPTERS 23기 Codex 스터디의 목표는 기능 체험에서 끝나지 않고, 실제 업무에 적용한 작은 사례를 매주 하나씩 남기는 것이다.
- 4주 흐름: 첫 사용·기능 지도 → Skill·Plugin·MCP → Browser·Computer Use 등 화면 기능 → 에이전트 시스템 설계.
- 운영 기준: AI는 초안을 만들고, 사람은 수치·권리관계·광고 표현·개인정보를 최종 검수한다.

## 3. 기존 실행 자산

- 운영 로드맵: **도시아재’s Codex 실무 적용 로드맵**
- 대시보드: <https://gpters23-codex-dashboard.vercel.app/output/study-dashboard/>
- 대시보드는 4주 목표, 자동화 후보, 실행·검증 루프, 증거 기록, 리스크 관리 기준을 담고 있다.

## 4. 1주차 자동화 업무 선정 기준

1. **작게 시작한다**: 한 번에 끝나는 작은 단위부터 실험한다.
2. **자주 하는 일을 고른다**: 반복할수록 시간 절감 효과가 쌓이는 업무를 우선한다.
3. **결과물이 명확한 일을 고른다**: 성공 여부를 바로 확인할 수 있어야 한다.
4. **사람 검수 후 안전하게 쓸 수 있어야 한다**: 특히 수치, 권리관계, 고객 안내 표현은 최종 검수한다.

### 첫 실험 후보

매물 관련 자료 1건을 받아 핵심 정보와 추가 확인 항목을 구조화한 **내부 브리핑 초안**을 만든다.

```text
입력 자료 1건
→ Codex 내부 초안
→ 원문 수치·권리관계·표현 검수
→ 수정된 최종본
```

### 성공 기준

- 핵심 정보와 추가 확인 항목이 구조화되어 있다.
- 원문 대조가 가능한 형태로 정리되어 있다.
- 사람이 검수하면 안전하게 사용할 수 있는 초안이다.
- 첫 결과물을 고객에게 바로 보내는 완성본으로 쓰지 않는다.

## 5. 컨텍스트와 스레드 운영 원칙

- Codex의 컨텍스트는 대화 내용, 실제로 읽은 파일, 정한 목표, 도구 실행 결과, 프로젝트 지침이 합쳐진 정보 묶음이다.
- 같은 프로젝트의 서로 다른 스레드는 대화 원문을 자동으로 공유하지 않는다.
- 대화는 즉시 쓰는 작업 기억이고, 파일·지침·대시보드는 오래 남기는 작업 기억이다.
- 새 스레드는 이 `HANDOFF` 파일과 `AGENTS.md`를 먼저 읽도록 지시한다.

## 6. 회사 PC 프로젝트 폴더 구조

```text
회사-PC-프로젝트/
├─ AGENTS.md
├─ HANDOFF-2026-07-22-gpters-codex-week1.md
├─ docs/
│  └─ study-notes/
│     └─ week1-automation-selection.md
├─ input/
│  └─ 매물-또는-공부서류-샘플/
├─ output/
│  └─ 내부-브리핑-초안/
└─ checklist/
   └─ 검수표.md
```

### 각 항목의 역할

- `AGENTS.md`: 항상 지켜야 할 규칙과 검수 원칙
- `HANDOFF-2026-07-22-gpters-codex-week1.md`: 오늘의 결정 사항과 다음 작업
- `docs/study-notes/week1-automation-selection.md`: 자동화 업무 선정 기준과 첫 실험 후보
- `input/`: 비식별화한 원본 자료 또는 샘플 자료
- `output/`: Codex 내부 초안과 검수 후 결과물
- `checklist/검수표.md`: 사람 최종 검수 기준

## 7. 회사 PC에서 할 일

1. 회사 PC에 위 폴더 구조를 만든다.
2. 이 `HANDOFF` 파일을 프로젝트 최상단에 저장한다.
3. `week1-automation-selection.md`를 `docs/study-notes/`에 저장한다.
4. 새 Codex 스레드를 만들고 아래 요청으로 시작한다.

> `HANDOFF-2026-07-22-gpters-codex-week1.md`와 `AGENTS.md`를 먼저 읽어 주세요. 현재 결정사항과 다음 할 일을 5줄로 확인한 뒤, 첫 실험을 위한 입력 자료 목록·내부 브리핑 출력 양식·사람 검수표 초안을 제안해 주세요.

## 8. Windows PowerShell로 폴더 구조 만들기

프로젝트 최상단 폴더에서 아래 명령을 실행한다.

```powershell
New-Item -ItemType Directory -Force -Path `
  'docs\study-notes', `
  'input\매물-또는-공부서류-샘플', `
  'output\내부-브리핑-초안', `
  'checklist'
```

## 9. Codex 권한 설정 실험 메모

현재 개인 PC의 전역 `config.toml`에는 스터디 실습을 위해 아래 설정을 추가했다.

```toml
approval_policy = "on-request"
approvals_reviewer = "auto_review"
sandbox_mode = "danger-full-access"
```

- 이 설정은 넓은 파일·네트워크 권한을 줄 수 있으므로, 민감 자료가 있는 회사 PC에 동일하게 적용할 필요는 없다.
- 문제가 있거나 불편하면 개인 PC의 백업 파일 `config.toml.before-study-permissions-20260722.bak`을 기준으로 되돌린다.
