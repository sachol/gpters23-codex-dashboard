# Week 1 과제 제출본

## 자동화하고 싶은 업무

제가 자동화하고 싶은 업무는 `공부서류 기반 매물 브리핑 초안 작성`입니다.

공인중개사 업무에서는 매물 상담이나 투자 검토 전에 건축물대장, 등기부등본, 토지이용계획, 실거래가, 현장 메모를 각각 확인하고 고객에게 설명할 수 있는 형태로 정리해야 합니다. 이 과정은 반복적이지만 수치와 법적 표현의 정확성이 중요해서 단순 문서 작업처럼 처리하기 어렵습니다.

## 현재 처리 방식

현재는 자료를 각각 열어 확인한 뒤, 중요한 수치와 권리관계, 용도지역, 임대 현황, 현장 장단점을 직접 메모합니다. 이후 고객에게 설명할 문구와 중개사 내부에서 확인해야 할 체크리스트를 따로 정리합니다.

현재 처리 흐름은 다음과 같습니다.

1. 주소와 거래 조건을 확인합니다.
2. 건축물대장에서 면적, 용도, 층수, 사용승인일을 확인합니다.
3. 등기부등본에서 소유자, 근저당권, 압류·가압류 등 권리관계를 확인합니다.
4. 토지이용계획에서 용도지역과 제한사항을 확인합니다.
5. 실거래가나 주변 시세를 확인합니다.
6. 현장 메모와 고객 질문을 정리합니다.
7. 고객 설명용 문구를 작성합니다.
8. 내부 검수 사항을 다시 확인합니다.

## 현재 소요 시간

아직 실제 1건 기준으로 실측하지 않았습니다. Week 1 전까지 같은 유형 업무 1건을 기존 방식으로 처리하며 시간을 측정할 예정입니다.

측정할 항목은 다음과 같습니다.

| 단계 | 측정 내용 |
|---|---|
| 자료 확인 | 공부서류와 시세 자료 확인 시간 |
| 수치 정리 | 면적, 가격, 용도, 임대 현황 정리 시간 |
| 브리핑 작성 | 고객 설명용 초안 작성 시간 |
| 내부 검수 | 권리관계, 공부서류, 광고 표현 재확인 시간 |

## Codex에게 맡기고 싶은 부분

Codex에게 최종 판단을 맡기기보다는, 사람이 검수하기 쉬운 초안과 체크리스트 작성을 맡기고 싶습니다.

| Codex에게 맡길 일 | 사람이 검수할 일 |
|---|---|
| 입력자료 목록화 | 원문 공부서류와 실제 일치 여부 확인 |
| 핵심 수치 추출 | 면적, 용도, 가격, 권리관계 최종 검수 |
| 누락·충돌 항목 표시 | 법적 판단과 고객 전달 가능 여부 판단 |
| 고객 설명용 초안 작성 | 과장 광고, 수익 보장 표현 제거 |
| 내부 검수 체크리스트 생성 | 고객 상담 전 최종 확인 |

## 왜 이 업무를 선택했는가

이 업무는 제 본업과 AI 중개실무 강의 활동에 모두 연결됩니다. 실제로 반복하는 업무이면서, 다른 공인중개사분들에게도 “Codex 앱을 이렇게 업무에 붙일 수 있다”는 사례로 보여주기 좋습니다.

또한 부동산 업무에서는 AI가 만든 답변을 그대로 쓰는 것이 아니라, 수치와 법적 표현을 검수할 수 있는 구조가 중요합니다. 그래서 이번 스터디에서는 Codex가 초안을 만들고, 공인중개사가 최종 판단과 검수를 맡는 방식의 에이전트 시스템을 만들어보고 싶습니다.

## Week 1 이후 실험 계획

1. 비식별 샘플 매물 자료로 브리핑 초안 생성 흐름을 테스트합니다.
2. 실제 업무 1건을 기존 방식으로 처리하며 시간을 측정합니다.
3. 같은 유형 업무를 Codex 지시서로 처리해 비교합니다.
4. 줄어든 단계와 여전히 사람이 검수해야 하는 단계를 기록합니다.
5. 최종적으로 GPTERS 사례글과 발표 자료로 정리합니다.

## 제출용 압축 문장

제가 자동화하고 싶은 업무는 `공부서류 기반 매물 브리핑 초안 작성`입니다. 현재는 건축물대장, 등기부등본, 토지이용계획, 실거래가, 현장 메모를 각각 확인한 뒤 고객 설명용 문구와 내부 검수 메모를 직접 정리합니다. 이 업무는 반복적이지만 수치와 법적 표현의 정확성이 중요해서 시간이 많이 걸립니다. Codex 앱으로는 입력자료 목록화, 핵심 수치 추출, 누락·충돌 항목 표시, 고객 설명 초안 작성, 내부 검수 체크리스트 생성을 맡겨보고 싶습니다. 최종 판단과 고객 전달 전 검수는 공인중개사가 직접 수행하는 방식으로 설계하려고 합니다.
