# 매일 실행 루틴

## 기본 원칙

하루 목표는 큰 자동화 완성이 아니라 `증거가 남는 30분 실험`입니다. 매일 같은 구조로 기록하면 4주 뒤 사례글과 발표 자료를 따로 기억해서 만들 필요가 줄어듭니다.

## 30분 기본 루틴

| 시간 | 할 일 | 결과물 |
|---|---|---|
| 5분 | 오늘 다룰 반복 업무 1개 선택 | 업무명, 오늘 목표 |
| 10분 | 기존 방식 또는 샘플 자료 정리 | 입력자료, 기존 처리 단계 |
| 10분 | Codex에게 작은 단위로 맡기기 | 초안, 표, 체크리스트, 프롬프트 |
| 5분 | 결과 검수와 기록 | 소요 시간, 배운 점, 다음 할 일 |

## 매일 시작 프롬프트

```text
오늘 GPTERS 23기 Codex 앱 스터디 목표를 이어서 진행해줘.
현재 폴더의 문서를 기준으로 오늘 30분 안에 할 수 있는 실험 1개를 정하고,
결과가 최종 사례글에 들어갈 수 있도록 소요 시간, 기존 방식, Codex 사용 방식, 배운 점을 기록해줘.
```

## 매일 종료 프롬프트

```text
오늘 한 작업을 GPTERS 사례글 재료로 정리해줘.
형식은 날짜, 사용한 Codex 기능, 입력자료, 결과물, 기존 방식 대비 차이, 실측 시간, 배운 점, 내일 할 일로 해줘.
부동산 관련 수치나 법적 판단은 단정하지 말고 확인 필요 항목으로 분리해줘.
```

## 기록 품질 기준

- `빨라졌다` 대신 `몇 분에서 몇 분으로 줄었는지`를 기록합니다.
- `잘 됐다` 대신 `어떤 단계가 자동화되었고 어떤 단계는 사람이 검수했는지`를 적습니다.
- `AI가 분석했다` 대신 `어떤 입력자료를 어떤 출력자료로 바꾸었는지`를 적습니다.
- 법적 판단, 권리관계, 수익성은 `확인된 사실`, `추정`, `추가 확인 필요`로 분리합니다.
