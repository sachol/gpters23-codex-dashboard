# 자동화 후보 정리

## 후보 평가 기준

| 기준 | 의미 |
|---|---|
| 반복성 | 매주 또는 매월 반복되는가 |
| 자료 구조화 | 입력자료가 파일, 링크, 표, 메모 형태로 정리 가능한가 |
| 검수 가능성 | Codex 결과를 사람이 빠르게 검수할 수 있는가 |
| 강의 활용성 | 다른 공인중개사에게 사례로 보여주기 좋은가 |
| 법적 리스크 | 단정 표현을 피하고 확인 항목으로 분리할 수 있는가 |

## 후보 목록

| 후보 | 현재 방식 | 기대 자동화 | 추천도 |
|---|---|---|---|
| 공부서류 기반 매물 브리핑 초안 | 건축물대장, 등기부, 토지이용계획, 실거래가, 현장 메모를 따로 확인해 브리핑 작성 | 입력자료 요약, 체크리스트, 고객 설명문, 리스크 확인 항목 생성 | 높음 |
| 고객 문의 상담 요약 및 후속 안내 | 통화/카톡 내용을 기억해 수동 정리 | 상담 요약, 니즈 분류, 후속 문자 초안, 추천 매물 조건 정리 | 높음 |
| 투자 검토 리포트 초안 | 가격, 임대료, 수익률, 주변 입지 자료를 따로 정리 | 수익률 계산표, 장단점, 확인 필요 사항, 브리핑 문구 생성 | 중간 |
| 블로그/강의 콘텐츠 재가공 | 실무 경험을 글, 카드뉴스, 강의안으로 수동 변환 | 사례글, 블로그 초안, 강의 슬라이드 목차 생성 | 중간 |
| 매물 홍보 문구 생성 | 매물 특징을 광고 문구로 수동 작성 | 고객군별 홍보 문구, 금지 표현 검토, 채널별 문안 생성 | 중간 |

## 1차 추천 주제

`공부서류 기반 매물 브리핑 초안 자동화 에이전트`를 1차 주제로 추천합니다.

이유:

- 공인중개사 실무의 전문성이 잘 드러납니다.
- 기존 `/부동산-매물-브리핑` 스킬과 연결하기 좋습니다.
- 건축물대장 API 교차검증, 등기부 확인, 토지이용계획 확인처럼 검수 기준을 명확히 세울 수 있습니다.
- 최종 사례글에서 Before/After 시간이 잘 보입니다.
- 강의 수강생인 다른 공인중개사에게 바로 설명할 수 있는 사례가 됩니다.

## 안전 기준

- 법적 권리관계, 위반건축물 여부, 용도지역, 면적 등은 `확정`이 아니라 `확인 결과`와 `추가 확인 필요`로 구분합니다.
- 건축물대장 API 결과는 단독 근거로 쓰지 않고, 원문 서류 또는 다른 공적 자료와 교차검증합니다.
- 고객에게 전달하는 문구는 투자 수익을 보장하지 않습니다.
- 광고 문구에는 과장 표현과 단정적 수익 표현을 넣지 않습니다.
