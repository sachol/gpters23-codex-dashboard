## 가져올 점

| 요소 | 이전 기수 장점 | 이번 23기 적용 |
|---|---|---|
| 후보를 먼저 보여주는 대시보드 | 11개 시나리오를 한 화면에서 보여줘 첫주 계획이 명확함 | 중개업무 후보와 강의준비 후보를 대시보드에 배치 |
| 카테고리 구분 | 콜라 단독, n8n 연동, 향후 확장으로 난이도 차이를 보여줌 | 즉시 실행, 실무 확장, 강의자료 확장으로 구분 |
| 시스템 아키텍처 | 데이터소스, n8n, 에이전트, 전달채널 흐름이 명확함 | 공부서류/상담메모 -> Codex -> 공인중개사 검수 -> 고객자료/사례글 |
| 카드형 시나리오 | 각 자동화 아이디어의 목적과 효과를 빠르게 이해 가능 | Week 2~4 사례 후보를 카드로 표시 |
| 구현 진행률 | 현재 0/11처럼 진행 상태가 직관적임 | Week 1 미니사례, Week 2~4 사례 진행률로 표시 |
| 프롬프트 예시 | 바로 따라 말할 수 있어 실행 장벽이 낮음 | 첫 실행 프롬프트를 복사 가능한 형태로 제공 |

## 그대로 가져오지 않을 점

| 요소 | 이유 |
|---|---|
| 지나치게 많은 시나리오 | 이번 기수는 4주 안에 실제 반복업무 사례를 만드는 것이 핵심이므로 3개 축으로 압축 |
| 완전 자동화 표현 | 부동산 업무는 수치, 권리관계, 광고 표현 검수가 필요하므로 사람 최종 검수 전제로 설계 |
| OpenClaw/n8n 중심 구조 | 이번 기수는 Codex App 기능 학습과 업무 자동화 사례 축적이 중심 |

## 이번 대시보드 개선 방향

1. 첫주에는 `무엇을 할지 정한 사람`처럼 보이게 만든다.
2. 2~4주에는 매주 `실제 업무 1건을 해본 사람`처럼 기록한다.
3. 최종에는 `공인중개사용 AI 중개실무 강의로 재사용 가능한 시스템`처럼 정리한다.
